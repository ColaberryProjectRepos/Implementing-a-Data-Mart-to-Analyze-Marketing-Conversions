6. Designing the Dimensional Model


Designing Dimensional Model to suit requirements of users which must address business needs and contains information which can be easily accessible. Also, the design of model should be easily extensible according to future needs. This model design must support OLAP cubes to provide prompt query results for analysts.



           Creating  Star/Snowflake Schema

A Snowflake Schema is an extension of a Star Schema, and it adds additional dimensions. It is called snowflake because its diagram resembles a Snowflake.

      Difference Between Fact Table and Dimension Table

            Fact Table:

                      A fact table is a primary table in a dimensional model.

            A Fact Table contains

                          -    Measurements/facts

                                -    Foreign key to dimension table

            Dimension table:

                                -    A dimension table contains dimensions of a fact

                                -    They are joined to fact table via a foreign key

                               -    Dimension tables are de-normalized tables

                                -    The Dimension Attributes are the various columns in a dimension table

                               -     Dimensions offers descriptive characteristics of the facts with the help of their attributes

                               -     No set limit set for given for number of dimensions

                               -     The dimension can also contain one or more hierarchical relationships
